Role × Functionality Matrix

2026-05-11 · TPG Partners · FOR KEN VALIDATION · What each role can do in the new VRS, mapped to the role-lens / slider-panel IA

This matrix describes the new VRS landing-page IA in terms of what each role does and sees. The shell is uniform — a bubble-field canvas with slider panels left and right — but the contents differ by role: which tabs appear, what notices surface, what context-menu actions exist on a vendor bubble, and what authority the user has during the period rhythm. Ken — we'd appreciate you eyeballing this and flagging anything wrong, missing, or simplified to the point of distortion. The roles below come from your AP-Side Overview plus the role-lens framing in VRS_Role_Lens_Vera; the SVP column is a guess based on a single column appearing in extracts.

Open questions for Ken

These are the spots where we're guessing — please correct or confirm:
  • SVP role. Column appears in some MDSE-extract dumps above DMM. Is there actually an SVP-level approval step, or is that column unused / legacy / a corner-case override path?
  • AP-stage agreement approval. We have it on AP Manager by default with AP Analyst as conditional. Which is right? Does APA do this routinely, or only the manager?
  • Buyer renewal-notice visibility. We have buyers seeing 90-day renewal alerts. Is that something buyers track today, or is it AP-driven?
  • Read-Only scope. Should R/O see full vendor portfolio, or division-scoped? (Finance audit vs. exec dashboard suggests two sub-roles.)
  • Reassignment authority. Only AP Manager today? Or is there a peer-reassignment between analysts during heavy weeks?
  • "Add deduction (on-demand)". AP Analyst & AP Manager. Anyone else, or never anyone else?
  • Approval-queue items for AP Manager. Marked conditional — meaning APM sometimes acts on AP-stage approvals. Should that be primary instead?
  • 1010 Intelligence panel for all roles. We've granted universal access. Any concern about exposing 1010-derived analytics to read-only / vendor-adjacent roles?

Out of scope of this matrix

  • Vendor Portal users. They live in the separate Java/Tomcat app, not in this shell. Their action set (view own programs, submit agreements, view 1010-derived pace-to-target) is described in doc 09 §4 and unchanged by this matrix.
  • Service accounts. Calculate, Batch-Exec, S5S5_Calculate, QUEUE, Upload, Finalize — these run scheduled work; they don't have a UI session.
  • NSA & S5S5 subsystem-specific operators. If there are dedicated users for these subsystems (vs. a generalist AP Analyst handling them), please flag — we'd add a column.
